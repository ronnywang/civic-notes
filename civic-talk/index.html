專案構想

讓公共討論真的有用

一個實驗中的平台：用 AI 整理多方立場，讓一般人也能快速掌握議題全貌，而不只是選邊站。

問題

為什麼大多數公共討論沒有用？

想參與一個公共議題的討論，你通常需要先花幾個小時自己找資料——而且找到的往往都是同溫層已經認同的東西。

另一個問題是，各方的意見散落在 Facebook、新聞留言、論壇各處，沒有一個地方可以讓你看到「這個議題上，到底有哪些不同的聲音」。

結果就是：有在追議題的人越說越確定，沒在追的人乾脆不參與。兩種人之間的鴻溝越來越大。

civic-talk 想做的，是用 AI 降低「從零到能參與討論」的門檻。

設計

三段式流程

STEP 1
INFORM
彙整多方立場
建立素材庫
→
STEP 2
OPINION
引導你形成
有根據的意見
→
STEP 3
SYNTHESIS
呈現立場分布
而非結論

INFORM：先讓大家站在同一個資訊起跑點

收集這個議題上不同立場的來源——政府文件、NGO 聲明、新聞報導、社群發言——用 AI 整理成結構化的素材庫，標注每份素材的來源與立場。不過濾，讓讀者自己判斷。

OPINION：不是問你支不支持，而是問你在乎什麼

AI 透過情境式問題引導（「如果你發現自己的車輛行蹤被陌生人追蹤，你會……」），讓你在看完素材後形成有根據的意見，而不只是選邊站。

SYNTHESIS：立場地圖，不是投票結果

輸出的不是「幾比幾支持」，而是「哪些人在乎哪些面向」的分布地圖。原始意見全部公開，讓操縱行為可以被社群檢驗。

關鍵選擇

為什麼這樣設計？

  • 素材收集靠人工，不靠爬蟲 網路內容結構複雜，動態載入、登入牆、反爬蟲機制讓自動抓取成功率很低。平台改由志願者手動複製貼上頁面原文，三個人各貼一次，比對一致才採用——確保完整性，也讓每份素材有人為把關。
  • 平台本身不用 AI token Server 端不呼叫任何 AI API。AI 運算的部分（例如生成素材摘要）由志願者用自己的 AI token 完成。很多人每週都有用不完的 token，這個平台讓閒置的算力變成公共資源。
  • 中立 = 廣度，不是立場平衡 不同立場的素材都要納入，包含立場鮮明、可信度有爭議的來源。重點是讓讀者看到「這個議題上真實存在的各種聲音」，而不是平台替讀者決定哪些聲音值得聽。
  • 輸出立場地圖，不輸出結論 平台的角色是呈現，不是裁決。每個人看完之後的結論可以不同，那本來就應該是各自的判斷。
現況

目前做到哪裡了？

  • ✓ 議題管理後台（建立議題、設定狀態）
  • ✓ 使用者註冊登入（第一位註冊者自動為管理員）
  • ✓ 素材 URL 提交（任何人都可以提交，防重複）
  • ✓ 素材原文貼入（志願者手動貼入頁面全文）
  • → 三份原文比對核實、INFORM 生成與查驗流程
  • ○ OPINION 情境問卷
  • ○ SYNTHESIS 立場地圖

這是一個個人 side project，目前還在早期驗證階段。如果概念可行，會考慮捐給 vTaiwan 或 g0v 社群繼續推進。

參與

你可以做什麼

1

提交素材 URL

看到和議題相關的文章、聲明、報導，直接貼 URL 進來。不需要判斷可信度，有代表性就夠。

2

貢獻閒置 token

如果你有 Claude / ChatGPT 的 token 快要重置，可以用來幫忙貼入素材原文或生成 INFORM 摘要。
